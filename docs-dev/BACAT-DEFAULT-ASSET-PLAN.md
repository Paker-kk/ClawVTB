# BACAT 默认资产方案

## 目标

为陪伴型桌宠 MVP 提供一个稳定、统一、可品牌化的默认 3D 身体方案，同时保留用户导入本地 VRM 的扩展能力。

当前产品角色分层应固定为：

- BACAT = 品牌人格、交互语言、情绪逻辑、桌宠行为
- 默认身体 = 官方提供的 BACAT 形象
- 用户身体 = 本地导入的 VRM 替身

这意味着：

- 用户可以换身体，但不能换掉 BACAT 的人格壳
- 默认身体必须优先保证品牌辨识度，而不是先追求高复杂度
- MVP 第一阶段允许默认身体先用低成本原型，占位不等于最终美术

## 当前结论

当前代码已经落下 BACAT 3D 原型运行时，作为默认资产的第一阶段占位体。

建议把默认资产策略拆成三层：

1. 第 0 层：程序生成占位体
说明：已经实现，适合立即验证 3D 桌宠链路、状态机、说话、导入逻辑。

2. 第 1 层：官方 BACAT Base Avatar
说明：第一套正式默认身体，必须跟随产品发布。

3. 第 2 层：官方扩展皮肤或情绪变体
说明：用于后续活动、节日、联名或会员版本，不属于 MVP 必需项。

## 默认身体推荐路线

### 路线 A：VRoid Studio 快速原型

适合：

- 需要 1 到 3 天内拿到第一版可用官方角色
- 团队暂时没有成熟 3D 角色建模能力
- MVP 优先验证产品形态，不先卷美术极限

优点：

- 直接进入 VRM 生态
- 表情、骨骼、基础头发物理默认可用
- 与本项目技术栈天然兼容
- 迭代快，换发型、服装、配色成本低

缺点：

- 品牌独特性有限
- 容易有强烈的 VRoid 模板感
- 如果 BACAT 不是二次元人形，风格约束会很大

结论：

这是 MVP 最快拿到“官方默认身体”的路线，但只建议作为第一版正式体，不建议作为长期唯一方案。

### 路线 B：Blender + VRM Add-on 正式化

适合：

- 需要更强品牌辨识度
- BACAT 不想长得像普通 VTuber 模板角色
- 后续要做海报、宣传片、截图和活动物料统一

优点：

- 品牌外形控制力最高
- 可以做更像 mascot 的比例、头身比和配件
- 更适合把 BACAT 做成桌宠，而不是直播人类替身

缺点：

- 美术生产链更长
- Rig、表情和导出规范更容易出错
- 前期需要明确角色设定稿

结论：

这是 MVP 之后必须走的正式方案。建议在 MVP 期间同步准备，但不要阻塞第一轮发布。

### 路线 C：Ready Player Me 或通用 SaaS Avatar

适合：

- 做用户生成内容
- 做快速角色创建
- 做多平台统一身份入口

缺点：

- 默认身体品牌感最弱
- 产品心智容易从 BACAT 桌宠变成通用 Avatar 工具
- 资产控制权与风格一致性较差

结论：

不推荐作为 BACAT 默认资产路线。可以作为后续用户创建入口调研，但不要抢默认角色位置。

## MVP 默认资产建议

MVP 阶段采用双轨方案：

1. 代码层默认仍保留 BACAT 3D 程序原型
说明：这是当前已经落地的保底体，保证任何时候都有默认角色。

2. 美术层尽快补一个 BACAT Base Avatar v1
说明：优先建议用 VRoid Studio 做第一版正式默认体，再逐步迁移到 Blender 正式版。

具体执行建议：

- 本周：继续用程序原型推进交互、情绪、口型、导入链路
- 同期：产出 BACAT 角色设定稿
- 设定确认后：先做 VRoid 版 BACAT Base Avatar v1
- 第二阶段：重做 Blender 正式版 BACAT Base Avatar v2

## 默认资产技术标准

官方 BACAT Base Avatar 进入 MVP 前，至少满足以下约束：

### 几何预算

- 总面数优先控制在 40k 到 70k triangles 之间
- Windows 常驻桌宠优先轻量，不要按直播主模型标准堆高模
- 头发和配件尽量简化，避免透明面过多

### 纹理预算

- 贴图优先 2K，必要时主脸保留 2K，其余降到 1K
- 避免过多材质球，优先控制 draw call
- 透明材质数量尽量少，桌面常驻比单次展示更怕性能抖动

### VRM 规范要求

- 优先 VRM 1.0
- 必须有完整 humanoid bone mapping
- 必须有基础表情预设
- 需要支持 blink、happy、relaxed、sad、surprised 至少五类常用表情
- 嘴型最好具备 aa、ee、oh 相关口型支持

### 物理和约束

- 只保留必要的 spring bone
- 长发、飘带、耳饰最多保留一到两组主要物理链
- 桌宠模式优先稳定，少而精，不追求大幅摆动

### 视觉风格

- 必须一眼看出 BACAT，不像随机 VTuber 角色
- 屏幕小尺寸下头部和眼部识别度要高
- 轮廓优先，不要把细节堆在看不见的局部
- 桌面远看要有强辨识，不是贴脸看模型截图

## 当前技术栈建议

根据 2026 年仍活跃的生态，当前推荐：

- Three.js：继续作为底层渲染
- @pixiv/three-vrm：继续作为 VRM 运行时主轴
- VRMA：作为通用动作资产格式
- MediaPipe Tasks Vision：保留为后续面捕和手势入口

不建议：

- 用 Ready Player Me 作为默认官方身体来源
- 让默认资产完全依赖在线生成器
- MVP 第一阶段就上重面捕和复杂控制器

## 产品层约束

默认资产必须满足一句话传播：

这是 BACAT，在桌面上陪你说话、思考和互动。

如果默认角色看起来只是一个普通 VRM 模型，那默认资产就是失败的。
