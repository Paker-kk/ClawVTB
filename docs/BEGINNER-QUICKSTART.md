# 小白快速开始

这份指南只服务一种人：

你不想研究架构，不想看一堆配置，也不想先学命令行。

你只是想做三件事：

1. 下载项目
2. 下载一个 VRM 模型
3. 让桌宠尽快跑起来

## 先说清楚

当前版本的桌宠主对话链路，仍然基于 OpenClaw Gateway。

这意味着：

- VS Code Copilot 和 Codex 现在可以通过向导帮助你检查和安装
- 但它们当前还不是桌宠的直接聊天后端
- 如果你只是想先把桌宠跑起来，核心还是把 OpenClaw 和 Gateway 配好

如果你要的是“桌宠直接调用 Copilot / Codex / Claude Code 当大脑”，这是下一阶段的多后端适配工程，不是本轮新手向导已经完全覆盖的部分。

## 最省事路线

### 第一步：准备三个东西

你需要先准备：

- 这个项目源码
- 一个 VRM 文件
- Node.js 18+

如果你还没有 Node.js，先去官网安装：

- https://nodejs.org/

## 第二步：启动项目

在项目目录执行：

```bash
npm install
npm start
```

首次启动时，会自动打开 Setup Wizard。

## 第三步：在 Setup Wizard 里直接做这几件事

### 环境预检页

现在向导会额外帮你检查：

- VS Code 是否已安装
- GitHub Copilot / Copilot Chat 是否已安装
- Codex CLI 是否已安装
- 当前有没有可用的 VRM 身体

你可以在这里直接点：

- 一键安装 Copilot
- 一键安装 Codex
- 选择 VRM

### Gateway 配置页

这一步决定桌宠能不能真正对话。

如果你已经装过 OpenClaw：

- 向导会尽量自动检测 Token 和端口

如果你没装：

- 先在环境页安装 OpenClaw
- 再启动 Gateway

### AI 模型页

这一步是给 OpenClaw 配模型，不是给 VS Code 配插件。

如果你只是想先能用：

- 选一个你已经有 API Key 的提供商
- 保存并测试

### 显示与完成页

走完整个向导后，桌宠就具备：

- 基础对话
- 语音播报
- 歌词显示
- VRM 身体加载

## VRM 怎么放最省心

有两种方式：

### 方式 A：在向导里直接选文件

这是最适合小白的方式。

- 点“选择 VRM”
- 选你下载好的 `.vrm` 文件
- 向导会把它记到配置里

### 方式 B：放到项目目录

也可以把 VRM 文件放到：

```text
models/vrm/
```

如果你没有手动导入其他 VRM，3D 模式会优先读取这个目录里的默认 VRM。

## Copilot 和 Codex 到底在这里扮演什么角色

### GitHub Copilot

适合：

- 在 VS Code 里写代码
- 用 Agent / Chat 帮你改项目
- 以后和桌宠做“编辑器陪伴联动”

### Codex CLI

适合：

- 在终端里直接做代码任务
- 跟你本地项目一起工作
- 以后和桌宠做“终端状态联动”

### 当前版本的现实情况

它们现在是“辅助开发工具安装目标”，不是“桌宠主聊天内核”。

别把这两件事混了。

## 你真正应该追求的第一次成功

第一次成功，不是把所有工具都装满。

第一次成功应该是：

1. 桌宠成功启动
2. Gateway 可连接
3. 能发一条消息
4. 你的 VRM 能显示出来

只要这四件事成立，后面再补 Copilot / Codex / 多后端都来得及。

## 下一阶段会做什么

如果后面继续推进，合理顺序应该是：

1. 把 OpenClaw 入口继续傻瓜化
2. 抽象统一的后端适配层
3. 再把 Claude Code / Codex / VS Code Copilot 真正接成桌宠后端

这是工程顺序，不是口号。
